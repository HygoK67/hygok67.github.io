单向链表 Singly-Linked List 没有提供在某一个元素之前插入元素的实现，提供了在某一个元素之后插入一个元素的实现 SLIST_INSERT_AFTER，时间复杂度为 O(1)；也提供了删除某一个元素的实现 SLIST_REMOVE，由于不知道该元素的前驱元素，因此必须进行循环查找，时间复杂度为 O(n)

实验中实现的双向链表实现了在某一个元素之后/之前插入元素的实现，分别为LIST_INSERT_AFTER 和 LIST_INSERT_BEFORE ，其时间复杂度都是 O(1)。实验代码也提供了删除某一个元素的实现 LIST_REMOVE，由于有双向的链接，因此实现了 O(1) 的删除。

对于循环链表，提供了元素的前/后插入实现，分别为 CIRCLEQ_INSERT_AFTER 和 CIRCLEQ_INSERT_BEFORE，由于循环链表的特殊结构，在插入时需要判断插入的元素有没有顶掉head/tail，如果有的话就需要及时进行更新。对于循环链表而言，其插入操作的时间复杂度同样为 O(1)。循环链表的删除实现 CIRCLEQ_REMOVE 复杂度为 O(1)，并且同样也需要判断是否删除的是循环的head/tail，如果是的话及时更新这两个field

Thinking 2.4

虚拟内存这个概念是相对于进程而言的，不同的进程应该有相互独立的虚拟内存。并且由于内存共享等需求，有可能会出现不同进程的相同虚拟地址映射到不同的物理地址以及不同进程的不同虚拟地址映射到同一物理地址的情况。为了解决多进程映射的这个问题，只通过虚页号是不够的，就必须引入能够标识当前虚拟地址所在地址空间的ASID。如果TLB中没有ASID的话，那么所有的进程在查询TLB时其实都是在查询一张页表，虚拟内存就失去了意义，相当于整个虚拟地址空间变成了和物理空间的一对一双射，进程之间的独立性也不存在了。因此mips 4KC的TLB中含有ASID字段是必要的

ASID段共8位，无预留，故总共可有 $2^8=256$​ 个不同的地址空间，但是由于TLB中还可以设置 G 位，绕开ASID，所以其实最终的地址空间共有 257 个

Thinking 2.6

对于x86体系架构来说，其和mips的差别主要体现在两点，其一在x86体系架构上由逻辑地址转换为物理地址的过程中中间会经历一步线性地址。逻辑地址首先会经过分段单元进行处理。每个分段包含一个起始地址和一个长度，用来定义一个内存段。通过段选择子和偏移量，逻辑地址被映射到一个线性地址。然后再由分页机制对该线性地址进行变换，得到物理地址。第二个不同点就出在变换到物理地址的过程中，在mips中，页表的维护、地址的变换以及tlb的填写都由软件来完成，而在x86体系结构中，tlb的填写以及地址的变换都由mmu硬件来完成，软件只需要对页表进行维护即可。

实验难点

本次实验中，积累了一个小知识，在此简单记录一下：

在以前的编程经验中，不论是CO课程还是c语言程序设计、数据结构，从来都是c语言是c语言、mips汇编是mips汇编，没有出现过两种语言混合在一个项目中的情况。但是本次在Lab2实验中，出现了例如 start.S 中的 _start 函数调用 init.c 中的 c 语言函数 mips_init 以及 tlbex.c 中的 c 语言函数 tlb_invalidate 调用 tlb_asm.S 中的 mips 汇编函数 tlb_out 这种两种语言互相调用的情况。
